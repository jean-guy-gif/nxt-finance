# V3.2 — Import Bilan — Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Permettre au dirigeant d'importer un bilan comptable (PDF/Excel ou template), de le valider et de le préparer pour l'analyse financière.

**Architecture:** Migration SQL (3 tables + ENUMs), service balance-sheet-service côté Next.js, appel parse-bilan via job-orchestrator, écran /analyse/import en 4 étapes visuelles, hooks React Query.

**Tech Stack:** Next.js 16 + Supabase + React Query 5 + Zustand + react-hook-form + Zod + Tailwind

## Task 1: Migration SQL — Tables bilan

**Files:**
- Create: `supabase/migrations/013_balance_sheets.sql`

**Step 1: Write the migration**

Create the migration with:

1. ENUMs:
   - `balance_sheet_source_type`: pdf_auto, excel_auto, template, manual
   - `balance_sheet_status`: uploaded, parsing, parsed, validating, validated, rejected
   - `balance_sheet_section`: actif_immobilise, actif_circulant, capitaux_propres, dettes, produits_exploitation, charges_exploitation, produits_financiers, charges_financieres, produits_exceptionnels, charges_exceptionnelles
   - `coherence_check_type`: actif_passif_balance, totals_consistency, missing_items, cross_period, duplicate
   - `coherence_check_status`: passed, warning, failed
   - `coherence_check_severity`: info, warning, critical

2. Table `balance_sheets`:
   - id UUID PK, agency_id FK→agencies CASCADE, fiscal_year INTEGER NOT NULL
   - source_type balance_sheet_source_type NOT NULL
   - source_file_path TEXT, overall_confidence DECIMAL DEFAULT 0 CHECK 0-100
   - status balance_sheet_status NOT NULL DEFAULT 'uploaded'
   - validation_notes TEXT, validated_by UUID FK→user_profiles, validated_at TIMESTAMPTZ
   - version_number INTEGER DEFAULT 1, is_current BOOLEAN DEFAULT true
   - parent_id UUID FK→balance_sheets self-ref, archived_reason TEXT
   - created_at, updated_at TIMESTAMPTZ DEFAULT now()
   - Partial UNIQUE: (agency_id, fiscal_year) WHERE status='validated' AND is_current=true
   - update_updated_at trigger

3. Table `balance_sheet_items`:
   - id UUID PK, balance_sheet_id FK→balance_sheets CASCADE
   - section balance_sheet_section NOT NULL
   - category TEXT NOT NULL, pcg_code TEXT
   - amount DECIMAL NOT NULL, amount_n_minus_1 DECIMAL
   - confidence_score DECIMAL DEFAULT 0 CHECK 0-100
   - is_validated BOOLEAN DEFAULT false, original_label TEXT NOT NULL
   - created_at, updated_at TIMESTAMPTZ
   - UNIQUE (balance_sheet_id, section, pcg_code) — only where pcg_code IS NOT NULL
   - update_updated_at trigger

4. Table `balance_sheet_checks`:
   - id UUID PK, balance_sheet_id FK→balance_sheets CASCADE
   - check_type coherence_check_type NOT NULL
   - status coherence_check_status NOT NULL, severity coherence_check_severity NOT NULL
   - expected_value DECIMAL, actual_value DECIMAL, message TEXT NOT NULL
   - created_at, updated_at TIMESTAMPTZ
   - update_updated_at trigger

5. Indexes: agency, fiscal_year, status, balance_sheet_id

6. RLS: SELECT, INSERT, UPDATE, DELETE for agency members (same pattern as V3.1)

Follow the exact SQL style from 012_v3_foundation_tables.sql.

## Task 3: Balance sheet service

**Files:**
- Create: `src/features/bilan/types.ts`
- Create: `src/features/bilan/services/balance-sheet-service.ts`

**Step 1: Create feature types**

Re-export relevant types from @/types.

**Step 2: Create balance-sheet-service.ts**

Functions to implement:

1. `fetchBalanceSheets(supabase, agencyId, fiscalYear?)` → BalanceSheet[]
2. `fetchBalanceSheet(supabase, id)` → BalanceSheet with items + checks joined
3. `createBalanceSheet(supabase, input: CreateBalanceSheetInput)` → BalanceSheet
   - Input: agencyId, fiscalYear, sourceType, sourceFilePath
   - Creates the balance_sheet row with status 'uploaded'
4. `validateBalanceSheetItem(supabase, itemId, correctedAmount?)` → BalanceSheetItem
   - Sets is_validated=true, updates amount if corrected
5. `validateBalanceSheet(supabase, id, validatedBy)` → BalanceSheet
   - Check: no checks with status='failed'
   - Check: overall_confidence >= BILAN_ANALYSIS_CONFIDENCE_GATE or all items validated
   - Set status='validated', validated_by, validated_at, is_current=true
   - Archive previous validated bilans for same agency+fiscal_year
6. `rejectBalanceSheet(supabase, id, reason)` → BalanceSheet
   - Set status='rejected', archived_reason
7. `runCoherenceChecks(supabase, balanceSheetId)` → BalanceSheetCheck[]
   - Delete existing checks for this bilan
   - Check actif_passif_balance: sum(actif_immobilise + actif_circulant) vs sum(capitaux_propres + dettes)
   - Check totals_consistency: verify section subtotals
   - Check missing_items: warn if key sections are empty
   - Upsert checks
   - Recompute overall_confidence
8. `uploadBilanFile(supabase, agencyId, file: File)` → string (file path)
   - Upload to Supabase Storage bucket 'bilans' under agency folder

Follow exact service patterns from commission-service.ts and job-orchestrator.ts.

4-step wizard with state machine:

```
UPLOAD → PARSING → VALIDATION → COMPLETE
```

State: `currentStep`, `balanceSheetId`, `jobId`, `balanceSheet`

**Step UPLOAD:**
- Choice: PDF/Excel libre or Template NXT Finance

- Show job progress via useJobStatus(jobId) polling
- Progress bar with percentage
- "Extraction en cours..." message
- When job completes: fetch the balance_sheet with items → advance to VALIDATION

**Step VALIDATION:**
- Display items grouped by section (actif, passif, produits, charges)
- Show confidence score per item (color coded: green >90, yellow 70-90, red <70)

- Allow validating individual items (checkbox)

- Success message

- Button: "Lancer l'analyse financière →" (link to /analyse/[id], future V3.3)

## Task 7: Wire parse-bilan Edge Function to balance-sheet-service

**Files:**
- Create: `src/features/bilan/services/parse-bilan-trigger.ts`

**Step 1: Create the trigger service**

This service is called AFTER the parse-bilan Edge Function completes (via job completion callback pattern). It:

1. Receives parsed items from the Edge Function response
2. Inserts them into balance_sheet_items
3. Runs coherence checks
4. Updates balance_sheet status from 'parsing' to 'parsed'

```typescript
export async function processParsedBilan(
  supabase: SupabaseClient,
  balanceSheetId: string,
  parsedData: { items: ParsedItem[], overallConfidence: number, warnings: string[] }
): Promise<void>
```

Also create a function to trigger the parse:
```typescript
export async function triggerBilanParsing(
  supabase: SupabaseClient,
  balanceSheetId: string,
  fileUrl: string,
  sourceType: BalanceSheetSourceType
): Promise<string> // returns jobId
```

This function:
1. Creates a processing_job via job-orchestrator
2. Updates balance_sheet status to 'parsing'
3. Calls supabase.functions.invoke('parse-bilan', { body: {...} })
4. On response: calls processParsedBilan
5. Completes the job
